# Claude Code ガイドライン

## 基本ルール

### README の更新

ファイルの追加・削除・変更を行った場合は、必ず `README.md` を更新すること。

更新が必要なケース:
- 新しいdotfileを追加した場合 → 「構成」テーブルに追記
- ファイルを削除した場合 → 「構成」テーブルから削除
- セットアップ手順が変わる場合 → 「セットアップ」セクションを更新
- 新しい依存関係が増えた場合 → 「依存関係」セクションに追記

### setup.sh の更新

新しい dotfile や設定ファイルを追加した場合は、必ず `setup.sh` を更新すること。

更新が必要なケース:
- `.config` 配下に新しいディレクトリを追加した場合 → `mkdir -p` コマンドを追加
- 新しいシンボリックリンクが必要な場合 → `link_file` 関数の呼び出しを追加
- ファイルを削除した場合 → 対応する `mkdir` と `link_file` を削除

**重要**: README の「構成」テーブルに追加したファイルは、必ず setup.sh にも反映すること。

### セキュリティ

- トークン、パスワード、APIキーなどの機密情報は絶対にコミットしない
- `.gitconfig` などに機密情報が含まれる場合は除去してからコミット
